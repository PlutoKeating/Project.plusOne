# Tutorial Style Guide — Skill

## 概述

本 skill 用于指导 AI 在为「加一的 Agent 成长之路」教程生成新页面时，维持原有的文本风格、叙事背景、知识点阐述逻辑与内容结构。

---

## 1. 叙事风格规范

### 1.1 主角设定

- **主角**：加一（plusOne）— 一个从零开始培养的 AI 助手
- **背景**：2050 年，深智科技公司为大型企业打造全方位智能助手
- **寓意**：加一的每一步成长，对应开发者需要掌握的技能

### 1.2 语言风格

- **亲切拟人化**：将技术概念拟人化，如"加一需要一具身体"、"加一睁开双眼"
- **避免学术腔**：使用口语化表达，贴近日常对话
- **故事驱动**：每个章节以"加一的成长故事"开篇，引出技术主题

### 1.3 示例表达

| 技术概念 | 拟人化表达 |
|---------|-----------|
| Python 基础 | 加一的身体 |
| LLM API 调用 | 加一睁开双眼 |
| RAG 系统 | 加一学会读书 |
| Agent 循环 | 加一学会动手 |
| Tool/MCP/Memory | 加一的工具箱 |
| Multi-Agent | 加一的团队 |

## 2. 知识阐述逻辑

### 2.1 核心结构：Why → What → How

```
┌─────────────────────────────────────────────────┐
│  为什么需要？（Why）                              │
│  ├─ 解决什么问题？                               │
│  └─ 没有它会怎样？                               │
├─────────────────────────────────────────────────┤
│  是什么？（What）                                │
│  ├─ 核心概念定义                                 │
│  └─ 与已有知识的关联                             │
├─────────────────────────────────────────────────┤
│  怎么用？（How）                                 │
│  ├─ 最小可行代码示例                             │
│  └─ 实践练习                                    │
└─────────────────────────────────────────────────┘
```

## 3. 代码示例规范

### 3.1 注释风格

- **中文注释**：所有注释使用中文，贴近目标读者
- **解释性注释**：说明"为什么这样做"，而非"做了什么"
- **变量命名**：使用有意义的英文变量名，避免拼音

### 3.2 示例代码结构

```python
# 你只需要知道这几种类型
name = "加一"             # 字符串 str
age = 0                   # 整数 int  
temperature = 0.7         # 浮点数 float

# print() 是你最好的调试朋友
print(f"加一的年龄: {age}")
```

### 3.3 代码块要求

- 使用完整的代码块，可直接复制运行
- 包含必要的 import 语句
- 关键行添加注释

## 4. 附加页面创建流程

### 4.1 文件位置与命名

```
docs/chXX/extra-NN-<slug>.md
```

- `XX`：章节编号（01-07）
- `NN`：附加页面序号（01, 02, 03...）
- `<slug>`：简短的英文描述，使用连字符分隔

**示例**：
- `docs/ch01/extra-01-virtual-env.md`（第1章的虚拟环境附加阅读）
- `docs/ch03/extra-02-vector-database.md`（第3章的向量数据库附加阅读）

### 4.2 创建步骤

#### 步骤 1：创建新文件

```bash
# 复制模板
cp docs/chXX/extra-template.md docs/chXX/extra-NN-<slug>.md
```

#### 步骤 2：填写内容

按照模板结构填写，确保：
- 标题包含 `[emoji] 附加阅读：[标题]`
- 关联章节信息正确
- 内容风格与主章节一致

#### 步骤 3：更新 _sidebar.md 与 所属章节的 README.md

在 _sidebar.md 对应章节下添加链接：

```markdown
- [🧱 第1章：Python 快速起步](ch01/)
  - [附加阅读：虚拟环境详解](ch01/extra-01-virtual-env.md)
```

在 所属章节的 README.md 中添加链接：

```markdown
## 📖 附加阅读
- [虚拟环境详解](ch01/extra-01-virtual-env.md)

## 📚 参考资料
...（原有内容保持不变）
```

#### 步骤 4：Git 同步

```bash
git add docs/chXX/extra-NN-<slug>.md docs/_sidebar.md docs/chXX/README.md
git commit -m "docs(chXX): add extra reading - <slug>"
git push
```

### 4.3 Commit 消息规范

```
docs(chXX): add extra reading - <slug>
```

**示例**：
- `docs(ch01): add extra reading - virtual-env`
- `docs(ch03): add extra reading - vector-database`

## 5. 内容质量检查清单

在发布新页面前，检查以下内容：

- [ ] 叙事风格与原教程一致（拟人化、亲切）
- [ ] 知识阐述逻辑清晰（Why → What → How）
- [ ] 代码示例可直接运行
- [ ] 注释使用中文
- [ ] 与主章节内容关联明确
- [ ] _sidebar.md 已更新
- [ ] Git commit 消息规范

## 7. 常见错误避免

| 错误类型 | 正确做法 |
|---------|---------|
| 使用学术腔 | 使用口语化、拟人化表达 |
| 缺少故事背景 | 以"加一的成长故事"开篇 |
| 代码无注释 | 关键行添加中文注释 |
| 知识点孤立 | 说明与主章节的关联 |
| 过度深入 | 保持"最小可行知识"原则 |
